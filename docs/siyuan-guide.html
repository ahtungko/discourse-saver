思源笔记配置指南

Discourse Saver V5.1 | 一键保存帖子到思源笔记

什么是思源笔记？

思源笔记 是一款开源本地优先的个人知识管理系统，支持块级引用和双向链接。
Discourse Saver 通过思源笔记的内核 API 将帖子内容保存为 Markdown 文档，自动转换为思源笔记的原生格式。
1 确保思源笔记已启动

插件通过思源笔记的本地内核 API 与思源通信，因此保存时必须保持思源笔记处于运行状态。

  1. 打开思源笔记桌面客户端
  2. 确认左下角状态栏显示内核已启动
  3. 默认 API 地址为 http://127.0.0.1:6806
提示：如果你修改了思源笔记的端口号，请在插件配置中同步修改 API 地址。
2 获取 API Token（可选）

如果你在思源笔记中开启了鉴权（Access Token），需要获取 Token 填入插件配置。

  1. 打开思源笔记，进入 设置 → 关于
  2. 找到 「API token」 项
  3. 复制 Token 值
字段说明示例
API Token思源笔记鉴权令牌token-xxxxxxxxxxxx
什么时候需要填 Token？
- 如果你没有开启鉴权（默认状态），可以留空
- 如果你开启了鉴权，则必须填写，否则会报权限错误
3 获取笔记本 ID

你需要指定帖子保存到哪个笔记本中。获取目标笔记本的 ID：

  1. 打开思源笔记
  2. 右键点击目标笔记本，选择 「设置」
  3. 在弹出的面板中找到 笔记本 ID
笔记本 ID 格式：
20210808180117-czj9bvb

特征：
以日期时间开头，格式为 YYYYMMDDHHMMSS-xxxxxxx
另一种获取方式：
打开思源笔记文档，在浏览器地址栏中，URL 格式为：
http://127.0.0.1:6806/stage/build/desktop/?...
也可以通过 API 获取：在浏览器访问 http://127.0.0.1:6806/api/notebook/lsNotebooks（POST 请求，body 为 {}）查看所有笔记本列表。
笔记本 ID 是必填项！
没有笔记本 ID，插件无法知道应该保存到哪里。
4 设置保存路径

指定帖子在笔记本中的保存位置。路径相对于笔记本根目录。

路径格式说明保存位置
/Discourse收集箱默认路径笔记本根目录下的「Discourse收集箱」文件夹
/收藏/论坛多级目录笔记本根目录 → 收藏 → 论坛
（留空）根目录直接保存到笔记本根目录
路径规则：
- 路径以 / 开头表示从笔记本根目录开始
- 不存在的文件夹会自动创建
- 帖子会自动按站点名称创建子文件夹（如 /Discourse收集箱/linux.do/）
5 配置浏览器插件
  1. 打开插件配置页面（右键插件图标 → 选项）
  2. 切换到 「思源」 标签页
  3. 在通用标签页中勾选 「保存到思源笔记」
  4. 填写配置信息：
配置项填写内容是否必填
API 地址思源笔记 API 地址，默认 http://127.0.0.1:6806必填
API Token步骤 2 获取的 Token（未开启鉴权可留空）可选
笔记本 ID步骤 3 获取的笔记本 ID必填
保存路径步骤 4 设置的路径，默认 /Discourse收集箱可选

填写完成后点击 「保存设置」。

6 测试连接
  1. 确保思源笔记正在运行
  2. 在插件配置页面点击 「测试连接」 按钮
  3. 成功后显示「连接成功」及笔记本信息
  4. 打开任意 Discourse 论坛帖子，单击链接按钮测试保存
配置完成！现在可以一键保存帖子到思源笔记了。
i 工作原理

了解插件如何将帖子保存到思源笔记：

环节说明
内容提取从 Discourse 帖子页面提取标题、正文、评论等内容
格式转换将 HTML 内容转换为 Markdown 格式
API 调用调用思源 /api/filetree/createDocWithMd 接口
自动转换思源笔记内核自动将 Markdown 转换为原生块格式
保存位置{笔记本}/{保存路径}/{站点名}/{帖子标题}.md
关于格式：
思源笔记的 createDocWithMd API 会自动将 Markdown 转换为思源原生的块引用格式。
标题、列表、代码块、图片、链接等 Markdown 语法都能正确渲染。

配置检查清单

常见问题

Q: 提示「连接失败」或「网络错误」
最常见的原因是思源笔记未启动。请确认：
1. 思源笔记桌面客户端正在运行
2. API 地址正确（默认 http://127.0.0.1:6806）
3. 如果修改过端口，需要同步修改插件中的 API 地址
4. 检查防火墙是否阻止了本地连接
Q: 提示「鉴权失败」(401/403 错误)
说明思源笔记开启了鉴权但 Token 不正确。
请到 设置 → 关于 → API token 重新复制，确保完整粘贴无多余空格。
Q: 提示「笔记本不存在」
检查笔记本 ID 是否正确。格式为 20210808180117-czj9bvb（日期时间-随机字符）。
确认目标笔记本没有被关闭或删除。在思源笔记中右键笔记本 → 设置 查看 ID。
Q: 保存后在思源笔记中找不到文档
1. 检查保存路径设置 — 文档保存在 {保存路径}/{站点名}/ 目录下
2. 在思源笔记中点击左侧文件树刷新
3. 使用思源笔记的全局搜索（Ctrl+P）搜索帖子标题
Q: 图片无法显示？
保存到思源笔记时，图片以原始 URL 链接形式嵌入。
如果图片显示不出来，可能是：
- 网络问题导致无法加载远程图片
- 图片来源站点有防盗链限制
建议：在 Obsidian 标签页开启「下载图片到本地」功能后，再手动复制到思源
Q: Markdown 格式在思源中显示正常吗？
正常！思源笔记的 createDocWithMd API 会自动将 Markdown 转换为思源原生块格式。
标题、列表、代码块、引用、表格、图片、链接等都能正确渲染。
少数高级 Markdown 语法（如 HTML 内联标签）可能会有差异。
Q: 可以同时保存到思源和其他平台吗？
可以！所有保存目标（Obsidian、飞书、Notion、思源、HTML导出）完全独立。
可以同时勾选多个保存目标，一键保存到多个平台。任何一个平台失败不影响其他平台。
Q: 思源笔记 Docker 部署如何配置？
如果你通过 Docker 部署了思源笔记：
1. 将 API 地址改为 Docker 映射的地址和端口
2. 确保浏览器能访问该地址
3. Token 同样在思源设置 → 关于中获取
注意：Docker 部署默认会开启鉴权，Token 为必填项
