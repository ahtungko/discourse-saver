Discourse Saver

保存 Discourse 论坛帖子到 Obsidian、飞书、Notion，或导出 HTML V5.5.0

关闭后需刷新页面生效
支持 Windows(D:\folder) / Mac(/Users/me/folder) / 相对路径
可以同时保存到多个平台
插件自动检测 Discourse 论坛，未被识别的站点可手动添加。
记录插件检测、注入、按钮拦截等运行状态，用于排查问题。最多保留500条，自动覆盖旧日志。

推荐留空，会自动保存到当前打开的 Obsidian 仓库
支持多种路径格式：智囊团/子目录、D:\Vault\文件夹、/Users/me/vault/folder
需先在 Obsidian 中安装 "Advanced URI" 插件
注意：启用后图片会以 Base64 格式嵌入 Markdown 文件中。优点是单文件完整保存，缺点是文件体积会显著增大。
注意：需要安装 Obsidian 社区插件「Local REST API」
默认 27124
媒体文件保存到 Vault 中此文件夹下，默认 media
从多维表格 URL 中获取：feishu.cn/base/bascnxxx?table=...

必须的权限：bitable:app（多维表格读写）
附件权限：drive:file:upload（可选）
字段：标题(文本)、链接(超链接)、作者(文本)、分类(文本)、标签(文本)、保存时间(日期)、评论数(数字)、正文(文本)、附件(附件)

在 Notion Internal Integrations 创建获取
从 Database 链接中复制 32 位 ID
⚠ 如果开启了 Base64 图片嵌入，Notion 将无法显示图片。建议关闭图片嵌入或仅用于 Obsidian。
属性名需与 Notion Database 中的列名完全匹配

配置步骤：
1. 创建 Internal Integration 并复制 Token
2. 创建 Database 并添加以上属性列
3. 在 Database 页面「...」→「Connections」→ 添加 Integration
4. 从 Database 链接中复制 ID

思源笔记本地 API 地址，默认 http://127.0.0.1:6806
在思源笔记 设置 → 关于 → 访问授权码 中设置，留空则关闭鉴权
可先留空点击「测试连接」，会列出所有笔记本及 ID
支持 /文件夹/子目录 格式，留空保存到笔记本根目录

配置步骤：
1. 确保思源笔记已启动（默认端口 6806）
2. 如需鉴权：设置 → 关于 → 设置访问授权码
3. 笔记本 ID 留空，先点「测试连接」查看可用笔记本列表
4. 复制目标笔记本 ID 填入上方输入框
5. 再次测试连接确认成功

在 yuque.com/settings/tokens 生成个人访问令牌
格式：用户名/知识库slug，可先点「测试连接」查看知识库列表

配置步骤：
1. 登录语雀，进入 设置 → Token 生成访问令牌
2. 先点「测试连接」查看知识库列表
3. 复制目标知识库的 namespace 填入上方
4. 再次测试连接确认成功

评论获取模式（三选一）：
条（0-10000）

⚠ 评论较多时可能需要较长时间

获取说明：
• ≤30条：从页面提取（快速）
• >30条或保存全部：通过 API 获取（完整）
• 超500条时显示加载进度

使用方法：
- 单击链接按钮 → 保存到各平台
- 双击链接按钮 → 复制链接
- Ctrl+Shift+S (Mac: ⌘+Shift+S) → 快捷键保存
