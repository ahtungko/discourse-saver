Notion Database 配置指南

Discourse Saver V5.3.1 | 一键保存帖子到 Notion

1 创建 Notion Integration
  1. 打开 Notion Internal Integrations
  2. 点击 「+ New integration」
  3. 填写 Integration 名称（如 Discourse Saver）
  4. 选择关联的 Workspace
  5. 点击 「Submit」 创建
  6. 创建成功后，复制 Internal Integration Secret
字段说明示例
Integration Token以 ntn_ 或 secret_ 开头ntn_xxx... 或 secret_xxx...
Token 格式说明：
• 2024年9月后创建的 Integration：Token 以 ntn_ 开头（新格式）
• 2024年9月前创建的 Integration：Token 以 secret_ 开头（旧格式）
两种格式都有效，插件均支持。
Token 非常重要，请妥善保存！不要分享给他人。
2 创建 Notion Database
  1. 打开 Notion，新建一个页面
  2. 输入 /database 选择 「Database - Full page」
  3. 输入 Database 名称（如 Discourse 收藏）
提示：也可以使用现有的 Database，只需确保有正确的属性类型。
3 配置 Database 属性

在 Database 中创建以下属性，类型必须匹配：

属性名（推荐）属性类型必填用途
标题Title（标题）✅帖子标题
链接URL✅帖子链接
作者Rich Text（文本）发帖人
分类Rich Text 或 Select分类
保存日期Date（日期）保存时间
评论数Number（数字）评论数
V4.0.2 更新：插件默认使用中文属性名！
按上表创建属性即可，无需额外配置。
如果使用英文属性名，请在插件设置中修改对应的属性映射。
类型必须正确！
- Title（标题）属性是每个 Database 必须有的，用于存储帖子标题
- URL 属性必须是 URL 类型，不是 Text
- Date 属性必须是 Date 类型
- Number 属性必须是 Number 类型
4 将 Integration 连接到 Database
  1. 打开你创建的 Database 页面
  2. 点击右上角 「...」 菜单
  3. 选择 「+ Add connections」
  4. 搜索并选择你创建的 Integration（如 Discourse Saver）
  5. 点击 「Confirm」 确认连接
这一步非常重要！
如果不将 Integration 连接到 Database，会出现 403 权限错误。
5 获取 Database ID

打开 Database 页面，从浏览器地址栏 URL 中提取 Database ID：

URL 格式解析：
https://www.notion.so/a1b2c3d4e5f67890a1b2c3d4e5f67890?v=xxxxxxxxxxxxxxxxxxxx

或者带有 Workspace 名称：
https://www.notion.so/workspace/a1b2c3d4e5f67890a1b2c3d4e5f67890?v=xxxxxxxxxxxxxxxxxxxx

✅ Database ID（要复制这个）：
a1b2c3d4e5f67890a1b2c3d4e5f67890（32位十六进制，在 ?v= 之前）

❌ View ID（不要复制这个）：
xxxxxxxxxxxxxxxxxxxx（?v= 后面的部分，是视图 ID，不是 Database ID）
常见错误：复制了 View ID 而非 Database ID
如果你的 URL 是 https://www.notion.so/3238a9c62f968048a2bbc2c82cc10e75?v=3238a9c62f9680bba9fb000c3163a72d
• ✅ 正确的 Database ID：3238a9c62f968048a2bbc2c82cc10e75
• ❌ 错误（View ID）：3238a9c62f9680bba9fb000c3163a72d
格式说明：
- Database ID 是 URL 中 ?v= 之前的 32 位十六进制字符串
- 带连字符的格式（如 a1b2c3d4-e5f6-7890-a1b2-c3d4e5f67890）也可以使用
- View ID 是用于保存表格视图设置的，不需要填入插件配置
6 配置浏览器插件
  1. 打开插件配置页面（右键插件图标 → 选项）
  2. 勾选 「保存到 Notion Database」
  3. 填写配置信息：
配置项填写内容
Integration Token步骤 1 获取的 Token（以 ntn_ 或 secret_ 开头）
Database ID步骤 5 获取的 32 位 ID
属性映射填写你 Database 中对应属性的名称
属性映射说明：
如果你的 Database 属性名是中文（如「标题」「链接」），
请在对应的映射框中填写中文名称，确保完全匹配。

点击 「保存设置」。

7 测试连接
  1. 点击 「测试连接」 按钮
  2. 成功后显示「连接成功」
  3. 打开 Discourse 论坛帖子，点击链接按钮测试保存
配置完成！现在可以一键保存帖子到 Notion 了。
关于评论保存的重要说明

由于浏览器技术限制，插件只能获取当前页面已加载的评论。

如果帖子评论较多：
1. 保存前请先滚动页面，加载更多评论
2. 插件只能抓取已经显示在页面上的评论
3. 未加载的评论（需要点击"加载更多"或滚动才能显示的）无法被获取

建议：保存前滚动到页面底部，确保所需评论已加载。

配置检查清单

常见问题

Q: 提示「Integration Token 无效」(401 错误)
检查 Token 是否正确复制，必须以 ntn_ 或 secret_ 开头。
• 2024年9月后创建的 Integration：Token 以 ntn_ 开头
• 2024年9月前创建的 Integration：Token 以 secret_ 开头
确认 Token 没有过期或被删除。如果不确定，可以重新创建一个 Integration。
Q: 提示「无权限访问该 Database」(403 错误)
最常见的原因是没有将 Integration 连接到 Database。
请参考步骤 4，在 Database 页面添加 Integration 连接。
Q: 提示「Database 不存在」(404 错误)
检查 Database ID 是否正确，应该是 32 位十六进制字符。
确认 Database 没有被删除或移动到其他 Workspace。
Q: 提示「属性映射错误」(400 错误)
检查插件中的属性名是否与 Database 中的属性名完全一致（区分大小写）。
确认属性类型正确：URL 属性必须是 URL 类型，Date 属性必须是 Date 类型。
Q: 提示「请求过于频繁」(429 错误)
Notion API 有速率限制。请稍等几秒钟后重试。
如果频繁出现此错误，请降低保存频率。
Q: 插件默认使用什么属性名？
V4.0.2 起默认使用中文属性名：
标题、链接、作者、分类、保存日期、评论数
按照教程创建中文属性名即可，无需额外配置。
如果你的 Database 使用英文属性名，请在插件设置中修改属性映射。
Q: 内容保存在哪里？
帖子内容会保存到 Notion 页面的正文区域（Page Content），
而不是保存到某个属性中。你可以在 Database 中点击任意条目查看完整内容。
Q: 和 Obsidian、飞书保存冲突吗？
不冲突！所有保存目标完全独立。
你可以同时启用 Obsidian、飞书、Notion、语雀和思源笔记，一键保存到多个平台。
任何一个平台保存失败不会影响其他平台。
